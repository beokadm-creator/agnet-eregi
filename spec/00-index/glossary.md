# 용어집(Glossary)

## 핵심 용어

- **사건팩(Case Pack)**: 등기 “업무 로직”을 제품화한 단위. 사건 정의/분기/서류/체크리스트/상태/SLA/리스크/가격/할당정책을 포함.
- **사건(케이스, Case Instance)**: 실제 사용자 의뢰 1건. 사건팩 + 입력값 + 문서 + 상태 + 로그로 구성.
- **뎁스(Depth Score)**: 업무 난이도 점수. 분기(관할 변경/외국인/긴급 등)에서 가산되며 할당/비딩/ETA에 영향을 줌.
- **리스크(Risk Score/Flags)**: 분쟁/반려/신원확인/문서 위조 가능성 등 운영 리스크(0~100 + 플래그). 일정 이상이면 manual_review.
- **Managed(중앙 할당)**: 플랫폼이 기본 정책에 따라 자동으로 수행자 배정(표준/저위험 케이스에 적합).
- **Restricted Bidding(제한 비딩)**: 후보 2~5곳만 초대해 제안(가격/시간/리스크/추가서류)을 받는 방식(비표준/고위험 케이스에 적합).
- **Manual Review(수동 검토)**: 운영자가 배정/진행을 직접 승인하는 모드(고리스크 안전밸브).
- **파트너(Partner)**: 입점 수행자(법무사사무소/법무법인/법률사무소). 실제 사건 수행 주체.
- **파트너 콘솔(Partner Console)**: 파트너가 PC로 케이스를 처리하는 업무용 화면(문서 슬롯/체크리스트/견적 확정/메시지).
- **운영 콘솔(Ops Console)**: 플랫폼 운영자가 수동검토/SLA/분쟁/정산/광고정책을 관리하는 화면.
- **SLA**: 착수/완료 등 기한 약속. 타이머로 관리하고 breach 시 에스컬레이션/재배정.
- **Value Preview Card**: 진단 단계에서 즉시 보여주는 가치 카드(예상 비용/시간/준비물). 이탈률 감소 핵심.
- **AI(편의 엔진)**: 법률 자문이 아니라 분류/요약/누락 탐지/초안 생성 역할. 확정 권한 없음(승인 게이트 필수).
