# 파일럿 실행 프로토콜 (corp_officer_change_v1)

작성일: 2026-04-17
대상: 파일럿 운영에 참여하는 파트너 및 내부 운영팀

---

## 1. 파일럿 개요
- **참여자**: 파트너 `p_demo_01` (테스트 계정 1명)
- **기간**: 2026-04-17 ~ 2026-04-24 (1주간)
- **목표 건수**: 최소 5건 ~ 최대 10건
- **성공 기준**:
  1. 완료율: 처리한 케이스 중 `completed` 도달 비율 (목표 100%)
  2. 처리시간: 1건당 평균 처리시간 측정 (목표 20분 이하)
  3. 에러/실패율: 치명적인 에러 발생 횟수 및 수동 복구 비율 측정

---

## 2. 파일럿 환경 및 계정
- **접속 URL**: `https://partner-console-eregi.web.app` (또는 제공된 배포 URL)
- **인증 방식**: 익명 로그인 (`익명 로그인` 버튼 클릭) 후 파트너 권한 주입 (`dev: partner 전환(p_demo_01)` 버튼 클릭)
- **접속 가이드**: [파일럿 온보딩 가이드](./2026-04-pilot-onboarding.md) 참조
- **리허설 스크립트**: [E2E 리허설 스크립트](./2026-04-e2e-rehearsal-script.md) 참조

---

## 3. 운영 체크포인트 및 모니터링
- **핵심 타임라인 이벤트 모니터링**:
  - `WORKFLOW_STAGE_CHANGED`: 단계 전진
  - `DOCUMENT_REVIEWED_NEEDS_FIX`: 문서 보완 요청
  - `SIGNATURE_TASK_COMPLETED`: 서명본 승인 완료
  - `PACKAGE_READY`: 제출 패키지 준비 완료
  - `CASE_STATUS_CHANGED`: 상태 변경 (특히 `completed`)
- **패키지 검증**:
  - `completed` 단계 도달 후 **[ZIP 포함 파일 검증]** 버튼을 클릭하여 서명본(`signed/`) 및 접수증이 정상적으로 Storage에 존재하고 패키지 포함 대상인지 확인합니다.

---

## 4. 수동 복구 프로토콜
- **네트워크/API 에러**: 화면 상단 또는 패널 내의 붉은색 에러 배너에서 **[다시 시도]** 클릭.
- **자동 전진 실패**: 타임라인이나 화면 상단 상태바의 '사유' 확인 후, 누락된 필수 서류나 서명본 보완. (수동 승인 필요 시 Ops 콘솔 개입)
- **다운로드/패키지 오류**: `PACKAGE_READY` 배지에서 다운로드 실패 시 운영팀에 문의하여 백엔드 스크립트로 수동 추출.
- 상세 복구 가이드: [파일럿 운영 체크리스트](./2026-04-pilot-ops-checklist.md) 참조

---

## 5. Known Issues (성능 경고)
- **번들 크기 경고**: `partner-console` 초기 로딩 시 500KB 이상의 청크(예: Firebase, React DOM 등)로 인한 경고가 있으나, 파일럿 기간 내 기능 동작에는 영향을 주지 않습니다. 파일럿 종료 후 Code Splitting 등 최적화를 진행할 예정입니다. (상세 내용: [성능 후속 조치](./2026-04-performance-followup.md))
