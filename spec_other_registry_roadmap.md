# ‘기타등기’(동산·채권담보 등) 확장 로드맵 — 사건팩 기반

## 1) 왜 ‘기타등기’는 사건팩 분리가 필수인가
동산·채권담보 등기는 **사건 유형/담보 목적/채무관계/첨부서류/제출 주체**에 따라 분기가 급격히 늘어납니다.  
따라서 “한 번에 모두”가 아니라, **Case Pack(사건팩) 단위로 잘라** 다음을 동시에 만족해야 합니다.

- (제품) 고객 입력 폼이 ‘필요한 것만’ 묻는다  
- (운영) 체크리스트/상태/SLA가 사건별로 다르다  
- (리스크) 분쟁/무효/서류위조/권리침해 리스크가 사건별로 다르다  
- (공급) 수행자(법무법인/법무사)의 전문 분야가 명확히 갈린다  

## 2) 추천 분류 체계(상위 도메인 → 사건팩)

### A. 동산·채권담보(담보등기)
- A1. 동산담보권 설정
- A2. 동산담보권 변경(담보목적물/채권자/채무자/담보범위)
- A3. 동산담보권 말소
- A4. 채권담보권 설정
- A5. 채권담보권 변경
- A6. 채권담보권 말소

### B. 기타 등록/공시(확장 시)
- B1. 가압류/가처분 관련(※ 리스크 높음: 별도 수동검토 라우팅 권장)
- B2. 법인 인감/제증명 발급 대행(등기 외 인접 영역)

## 3) ‘기타등기’ 전용 리스크 프레임(필수)
다음 조건 중 하나라도 걸리면 **assignmentPolicy.mode = manual_review**로 라우팅하는 것을 기본값으로 권장합니다.

- 채권 양도/담보권 이전이 포함됨(당사자/권리관계 복잡)
- 담보 목적물이 다수이거나 식별이 어렵다(목록/식별자/증빙 불명확)
- 이해관계자(채무자/제3취득자 등) 다수
- 긴급(마감 72h 이하) + 고액(금액/채권규모) 조합
- 분쟁/소송/가처분 진행 중

## 4) 사건팩 생성 가이드(템플릿 적용 순서)
1. **scope.includes/excludes**로 경계부터 확정  
2. **inputForm**: 법률적 필수요건을 충족시키는 최소 입력값 정의  
3. **decisionTree**: “난이도/리스크를 만드는 분기”를 먼저 정의(서류는 나중)  
4. **documents/checklist**: 분기별 서류와 검토 포인트를 매핑  
5. **workflow/sla**: 상태 전이와 SLA를 운영 가능하게 설정  
6. **scoring**: baseDepth/baseRisk + 분기 가산점으로 수치화  
7. **assignmentPolicy**: depth/risk로 managed vs restricted_bidding vs manual_review 규칙 작성

## 5) 6주 확장 플랜(권장)
- 1~2주차: A6개 중 “말소/변경”을 제외하고 **설정 1개(A1 또는 A4)**만 사건팩으로 출시  
- 3~4주차: 같은 도메인의 **변경/말소** 추가(서류/분기 재사용률이 높음)  
- 5~6주차: 분쟁 가능성이 큰 케이스를 manual_review로 흡수하는 “안전밸브” 강화 + 파트너 풀 확장
