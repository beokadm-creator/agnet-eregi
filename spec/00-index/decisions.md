# 결정사항 로그(Decisions)

본 문서는 “왜 이렇게 설계했는지”를 빠르게 추적하기 위한 결정사항 로그입니다.

## 2026-04-13

### 범위
- 등기 범위: **법인등기 + 기타등기(동산·채권담보 등)**  
  - 부동산등기는 초기 범위에서 제외(리스크/케이스 폭발 방지)

### UX 전략
- 사용자(비전문가)는 **모바일 퍼스트**: “구글처럼 단순한 입력(자연어 + 카테고리)”로 시작
- 파트너/운영은 **PC 퍼스트**: 다문서 업로드/검토/케이스 큐 처리 최적화

### 플랫폼 모델
- 공급자: **법무사사무소 + 법무법인/법률사무소** 모두 포함  
  - 영세 사무소의 “업무 OS 부재”를 플랫폼이 채움(락인/확장성)

### 매칭/가격
- “중앙 할당 vs 비딩”은 하이브리드:
  - 표준/저위험: 자동 할당(Managed)
  - 비표준/고위험: 제한 비딩(Restricted bidding) 또는 수동 검토(Manual review)

### 광고/추천
- 추천 영역을 2트랙으로 분리:
  - **스폰서(광고) 추천**: 라벨 고정, Verified/Pro만, 최소 품질 기준 강제
  - **품질 추천(비광고)**: 적합도/SLA/품질/가용성 기반
