**엔진 사용처 & 오케스트레이션 스펙 v1**

작성일: 2026-04-13

**1. 목표**

- 엔진(규칙+AI)을 ‘어디서/언제/무엇을 위해’ 호출하는지 확정하여 UX를 안정화한다.
- 사용자(모바일)는 단순 입력→최소 진단→추천/비교→사건 진행으로 이어지고, 파트너/운영(PC)은 케이스 처리에 최적화된다.
- AI는 ‘후보/추천/초안/경고’까지만. 확정(견적/발송/진행)은 승인 게이트로 통제한다.

**2. 엔진 구성(역할 분해)**

|                    |                                        |
| ------------------ | -------------------------------------- |
| **엔진**             | **역할**                                 |
| 인텐트/업무 라우터(AI)     | 자연어를 사건팩 후보(1~3)로 매핑 + 필수 질문 생성       |
| 사건팩 규칙 엔진          | 분기 적용 → 준비물/체크리스트/리스크/뎁스 계산            |
| 파트너 후보/정렬 엔진       | 필터링(역량/지역/가용성) + 랭킹(SLA/품질/가격)         |
| 견적 엔진              | 파트너 가격룰(기본가+가산)로 예상 견적 범위 산출(확정 금지)    |
| ETA 엔진             | 사건팩 기본값 + 분기 + 파트너 통계 + 용량으로 ETA 범위 산출 |
| 문서 대략 검토 엔진(AI+규칙) | 문서 슬롯 분류, 불일치/누락 후보, 보완 메시지 초안(승인 필요)  |
| 알림 엔진              | 다음 할 일/마감/SLA 기반 재진입 알림(푸시/카톡)         |

**3. 모바일(사용자) 화면별 엔진 사용처**

**3.1 홈(자연어/카테고리)**

- 호출: 인텐트/업무 라우터
- 출력 UI: 사건팩 후보 + 다음 질문(1번째 질문 카드로 전환)
- 캐시: 최근 입력/최근 선택(재진입)만 로컬 저장

**3.2 미니 진단(3~7문항)**

- 호출: 사건팩 규칙 엔진(매 문항 응답 시), ETA/견적(옵션: 입력이 충분해지면)
- 출력 UI: 준비물(임시), 예상 비용/시간 범위(임시), 다음 질문
- 가드레일: 정보 부족하면 ‘추가 질문’만(결론/확정 금지)

**3.3 결과(추천/비교 리스트)**

- 호출: 파트너 후보/정렬 엔진 + 견적 엔진 + ETA 엔진
- 출력 UI: 추천 1안(근거 3줄) + 비교 Top3 + 광고(라벨 고정)

**3.4 사건 워크스페이스(진행 OS)**

- 호출: 문서 대략 검토 엔진(업로드 이벤트), 알림 엔진(마감/다음 할 일)
- 승인: AI 보완 메시지는 파트너/운영 승인 후 발송
- 감사: 문서 버전/검토 결과/승인 내역 로그 필수

**4. PC(파트너/운영) 콘솔에서 엔진 사용처**

- 파트너 콘솔: 케이스 상세에서 문서 검토 엔진(초안) + 견적 엔진(범위) → 최종 확정은 ‘사람 클릭’ + 사용자 동의 로그.
- 운영 콘솔: 고리스크/분쟁/브리치 큐에서 ‘요약/권고’만 AI로, 실행(재배정/환불/중단)은 운영자.

**5. 승인 게이트(확정 금지 영역)**

|               |                                   |
| ------------- | --------------------------------- |
| **행위**        | **승인 규칙**                         |
| 보완 요청 메시지 발송  | AI 초안 → 파트너/운영 승인 후 발송(자동 발송 OFF) |
| 견적 확정(결제 전/후) | 파트너 또는 운영자 확정 + 사용자 동의 로그         |
| 고리스크 케이스 진행   | 운영자 수동 검토 후 라우팅(manual_review)   |
