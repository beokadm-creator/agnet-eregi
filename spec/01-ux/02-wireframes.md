# 텍스트 와이어프레임(요약)

본 문서는 개발자가 빠르게 구조를 잡을 수 있도록, “텍스트 와이어프레임”의 핵심만 요약합니다.  
상세 ASCII 와이어프레임은 레거시 산출물(`ux_wireframes_mobile_user_pc_console_v1.docx`)을 참조하되, 구현은 본 저장소의 MD/YAML을 단일 소스로 삼습니다.

## 1) 모바일 사용자(비전문가)

### Home (구글형)
- 단일 입력창(자연어 1줄)
- 카테고리 칩(법인등기 / 기타등기)
- 예시 쿼리(클릭)
- CTA 1개: “진단 시작”

### Mini Diagnosis (3~7문항)
- 1화면 1문항
- 동시에 “Value Preview(비용/시간/준비물)” 카드 고정 노출
- 자동 저장/재진입(딥링크)

### Results (추천+비교)
- “추천 1안” 카드(근거 1~3줄)
- “비교 Top3” 리스트 + 더보기
- “스폰서(광고)”는 별도 영역 + 라벨 고정

### Partner Detail (직방/다방)
- 지도/주소/영업시간/주차/예약
- 온라인 진행 CTA(Primary) + 방문 예약(Secondary)

### Case Workspace (진행 OS)
- 탭: 요약/준비물/업로드/진행/메시지/결제
- 서류는 슬롯 단위 상태(미업로드/검토중/OK/보완필요)
- AI 검토는 “초안/확정 아님” 표준 라벨

## 2) PC 파트너 콘솔(업무 처리)
- 케이스 큐(필터, SLA 타이머, 빠른 액션)
- 케이스 상세(문서 슬롯 + 체크리스트 + AI 초안 + 메시지 승인)
- 견적 확정(사람 확정 + 사용자 동의 로그)

## 3) 엔진 연동
- 화면/이벤트별 엔진 호출은 `../../spec_engine_orchestration.yaml`이 단일 소스입니다.
