# 문서 저장/암호화/보관/삭제(수명주기) 스펙 — v1

목표: 등기 플랫폼의 본질은 “문서”이므로, 문서 저장은 보안/운영/법무의 중심이다.  
운영을 AI에게 맡길수록 문서 접근 경로가 늘어나므로 **기본값을 ‘최소 접근 + 감사 + 보관정책’**으로 고정한다.

---

## 0) 문서 수명주기 정의(개념)

문서 객체(논리):
- `documents`: “주주명부/인감증명/정관” 같은 논리 문서(슬롯에 매핑)
- `document_versions`: 실제 업로드 파일의 버전(append-only)
- `document_ocr_results`: OCR/추출 결과(민감 가능)

핵심 원칙:
1) 파일은 **append-only 버전**으로만 저장(기존 버전 덮어쓰기 금지)
2) “삭제”는 소프트 삭제 + 물리 삭제(보관기간 종료 후) 2단계
3) 다운로드/열람은 항상 감사로그

---

## 1) 저장소(Storage) 요구사항

### 1.1 파일 저장 방식
권장: 오브젝트 스토리지(S3/GCS 등) + DB에는 참조키(`storage_ref`)만 저장.

필수:
- 서버 사이드 암호화(SSE) 또는 클라이언트 사이드 암호화(앱 레벨 KMS) 중 1개 이상
- pre-signed URL은 **짧은 TTL**(예: 60~300초)
- 파일 접근은 반드시 인증된 API를 통해서만(직접 공개 버킷 금지)

### 1.2 메타데이터(필수)
`document_versions`에 최소:
- `sha256` (중복 감지/무결성)
- `size_bytes`, `mime_type`, `file_name`
- `created_by_actor_type/id`

---

## 2) 업로드 파이프라인(악성/품질/PII 대응)

업로드 시 서버가 수행해야 하는 최소 단계:
1) 바이러스/악성 검사(필수)
2) MIME/확장자 검증(필수)
3) 크기 제한/페이지 제한(정책)
4) 해시 계산(sha256)
5) `DOCUMENT_UPLOADED` 이벤트 발행(append-only)

선택(권장):
- OCR/텍스트 추출 비동기 처리(`document_ocr_results`)
- 문서 분류/슬롯 추천(AI, 단 확정은 사용자/파트너)

---

## 3) 접근 제어(문서/버전/OCR)

### 3.1 기본 접근 규칙
- user: 자기 케이스 문서만
- partner: 자기 배정 케이스 문서만
- ops: 기본 조회 가능하나, **OCR/PII 계열은 별도 승인**

### 3.2 OCR/추출 결과는 “PII 취급”
`document_ocr_results`는:
- 기본 차단(ops도)
- `PII_VIEW` + 승인 게이트 + 사유 + 만료 시간으로만 열람
- 열람 시 `audit_logs`에 강제 기록

---

## 4) 보관/삭제/법적 대응(정책)

> 실제 법률 자문이 아니라 “제품 정책을 고정하기 위한 초안”입니다.  
> 보관기간은 서비스/법무 검토로 확정되며, 시스템은 “정책이 바뀌어도” 적용 가능해야 합니다.

### 4.1 보관기간(예시 정책)
- 진행 중 케이스: 종료 시점까지 보관
- 종료 후 보관: 예) 3년(분쟁/대사 대응) — **정책 값으로 관리**
- 광고/정산 증빙: 예) 5년(회계/세무) — 정책 값으로 관리

### 4.2 삭제 유형
- 사용자 삭제 요청(가능 범위 내): “접근 차단(soft)” → “보관기간 종료 후 물리 삭제”
- 법적 보존(Legal Hold): 분쟁/수사/소송 등으로 삭제 금지 플래그

권장 필드(예):
- `documents.legal_hold = boolean`
- `documents.retention_until = timestamptz`
- `documents.deleted_at = timestamptz`

### 4.3 물리 삭제(필수)
배치 작업:
- `retention_until < now()` 이고 `legal_hold=false`인 버전들을 대상으로
- 오브젝트 스토리지에서 실제 삭제
- DB에는 “삭제 완료” 이벤트/로그 남김(감사)

---

## 5) 다운로드/공유/외부 전송 통제

### 5.1 사용자/파트너 다운로드
- 다운로드 URL은 pre-signed + 짧은 TTL
- 대량 다운로드/스크래핑 방지(레이트리밋)
- 모든 다운로드는 `audit_logs`에 기록(최소: actor, docVersionId, ip, userAgent)

### 5.2 외부 전송(카카오/이메일 첨부 등)
원칙: “첨부파일 전송”은 위험하므로 링크 기반을 기본으로.
- 링크는 만료/회수 가능해야 함
- 민감 문서는 링크 접근 시 재인증 요구 가능

---

## 6) 문서 버전 정책(업무 품질/분쟁 대비)

필수:
- 버전은 append-only
- “OK”였던 문서라도 새 버전 업로드 시 `검토중`으로 돌아갈 수 있음(상태 머신 준수)

권장:
- 버전 간 diff(파일 레벨 diff는 어려우므로 메타/이슈 비교) 제공
- “어느 버전으로 제출했는지” 제출 증빙을 별도 저장(사건팩별)

---

## 7) AI 문서 사용 정책(프롬프트 인젝션/데이터 유출 방지)

원칙:
- 문서 내용은 “지시”가 아니라 “데이터”
- 모델 입력에는 최소 텍스트만(필요 슬롯만)
- PII 마스킹/토큰화 후 처리(가능한 경우)

필수 로그:
- 어떤 문서/버전을 어떤 엔진이 읽었는지 `audit_logs` 또는 `domain_events`로 남김
- 모델/정책 버전(재현)
