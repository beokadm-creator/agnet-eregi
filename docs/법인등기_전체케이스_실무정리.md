# 대한민국 법인 등기 전체 케이스 실무 정리

## 문서 범위
- 대상: 대한민국 `법인 등기` 전반
- 기준: 일반적인 `주식회사` 실무를 중심으로 정리하되, `유한회사/합명회사/합자회사/외국회사/비영리법인` 등은 별도 확인 필요 사항으로 표시
- 목적: 실제 등기 업무 처리가 가능할 정도로 `필수 수집 정보 + 프로세스 + 산출물 + 예외/리스크 + 퍼널 질문 설계`를 운영 문서화
- 주의:
  - 본 문서는 `통합 마스터`다.
  - 실제 접수는 케이스별로 `설립 / 변경 / 자본·주식 / 조직행위 / 해산·청산 / 특수` 하위 시나리오로 분리하는 것이 권장된다.
  - 일부 항목은 회사 형태, 정관, 이사회 설치 여부, 관할 등기소 실무에 따라 달라질 수 있으므로 `추정/불확실`로 표시했다.

- 기본 변경등기
  - 상호 변경
  - 목적 변경
  - 본점 이전
  - 공고방법 변경
  - 지점 설치/이전/폐지
  - 지배인 변경
- 임원/대표 관련
  - 이사/감사/대표이사 취임
  - 사임/해임/중임
  - 공동대표/각자대표 변경
- 자본·주식 관련
  - 자본금 증자
  - 자본금 감소
  - 주식분할/병합
  - 주식매수선택권, 전환사채, 신주인수권부사채 관련 변경
- 조직행위
  - 합병

  - 조직변경

  - 회사계속

- 특수 케이스
  - 외국회사 등기
  - 현물출자

  - 과거 반려/보정 이력 있는 재신청

## 1. 반드시 수집해야 하는 정보

### 1-1. 공통 필수 정보
아래는 거의 모든 법인 등기에서 빠지면 진행이 어려운 공통 정보다.

#### A. 법인 기본 정보
- 법인명(정확히)
  - 질문 문장: `법인등기부상 정확한 법인명을 입력해 주세요.`
  - 입력 타입: `text`
  - 예시: `주식회사 에이전트레지`
  - 검증 규칙: 등기사항전부증명서와 완전 일치, 공백/괄호/영문 병기 여부 확인
- 법인등록번호
  - 질문 문장: `법인등록번호를 입력해 주세요.`
  - 입력 타입: `text`
  - 예시: `110111-1234567`
  - 검증 규칙: 하이픈 포함/제외 모두 허용 후 저장 시 정규화, 자릿수 형식 확인
- 회사 형태
  - 질문 문장: `회사 형태를 선택해 주세요.`
  - 입력 타입: `single_choice`
  - 예시: `주식회사`
  - 검증 규칙: `주식회사 | 유한회사 | 합명회사 | 합자회사 | 외국회사 | 기타`
- 현재 본점 주소
  - 질문 문장: `현재 등기부상 본점 주소를 입력해 주세요.`
  - 입력 타입: `text`
  - 예시: `서울특별시 강남구 테헤란로 123, 10층`
  - 검증 규칙: 도로명주소 기준, 상세주소 포함 여부 확인
- 관할 등기소
  - 질문 문장: `현재 본점 기준 관할 등기소를 알고 있으면 입력해 주세요.`
  - 입력 타입: `text`
  - 예시: `서울중앙지방법원 등기국`
  - 검증 규칙: 주소 기반 추정값과 다르면 수동 검토
- 정관 최신본 보유 여부
  - 질문 문장: `최신 정관 파일을 보유하고 있나요?`
  - 입력 타입: `single_choice`
  - 예시: `있음`
  - 검증 규칙: `있음 | 없음 | 불확실`
- 최근 등기사항전부증명서 확보 여부
  - 질문 문장: `최근 발급한 등기사항전부증명서를 가지고 있나요?`
  - 입력 타입: `single_choice`
  - 예시: `있음`
  - 검증 규칙: `있음 | 없음`, 가능하면 발급일 입력 추가

### 1-2. 케이스 특화 필수 정보
전체 케이스를 한 번에 다루기 위해 `사건군`별 핵심 변수만 정리한다. 실제 운영에서는 각 사건군을 개별 시나리오로 분리하는 것이 바람직하다.

#### A. 설립
- 회사 형태
  - 질문 문장: `설립할 회사 형태를 선택해 주세요.`
  - 입력 타입: `single_choice`
  - 예시: `주식회사`
  - 검증 규칙: 법인형 필수 선택
- 상호
  - 질문 문장: `설립할 상호를 입력해 주세요.`
  - 입력 타입: `text`
  - 예시: `주식회사 에이전트레지`
  - 검증 규칙: 동일 관할 내 동일상호/유사상호 사전검토 필요
- 본점 예정지
  - 질문 문장: `설립할 회사의 본점 예정 주소를 입력해 주세요.`
  - 입력 타입: `text`
  - 예시: `서울특별시 성동구 ...`
  - 검증 규칙: 주소 정규화, 사용권원 확인 필요 여부 체크
- 목적
  - 질문 문장: `사업 목적을 입력해 주세요.`
  - 입력 타입: `multi_choice`
  - 예시: `소프트웨어 개발업, 전자상거래업`
  - 검증 규칙: 업종 인허가 필요 여부 검토
- 자본금
  - 질문 문장: `설립 시 자본금을 입력해 주세요.`
  - 입력 타입: `number`
  - 예시: `10000000`
  - 검증 규칙: 원 단위 정수, `원`
- 발행주식 총수 / 1주의 금액 / 설립 시 발행주식수
  - 질문 문장: `발행주식 총수, 1주의 금액, 설립 시 발행주식 수를 입력해 주세요.`
  - 입력 타입: `number`
  - 예시: `총수 10000주 / 1주 1000원 / 발행 10000주`
  - 검증 규칙: `발행주식수 x 1주 금액 = 자본금`
- 발기인 정보
  - 질문 문장: `발기인별 성명(법인명), 주소, 인수주식 수를 입력해 주세요.`
  - 입력 타입: `text`
  - 예시: `홍길동 / 서울시 ... / 6000주`
  - 검증 규칙: 총 인수주식 수 합계 일치
- 임원 구성
  - 질문 문장: `초기 임원 구성(이사, 감사, 대표이사)을 입력해 주세요.`
  - 입력 타입: `text`
  - 예시: `사내이사 2명, 감사 1명`
  - 검증 규칙: 회사 형태별 법정 요건 및 정관 일치 여부 확인
- 납입 정보
  - 질문 문장: `주금 납입일과 납입 계좌 정보를 입력해 주세요.`
  - 입력 타입: `date + text`
  - 예시: `2026-05-07 / 발기인 대표 계좌`
  - 검증 규칙: 설립 일정과 선후 관계 확인

## 2. 실무 프로세스

### Step 1. 사전 진단 및 등기유형 확정
- 등기사항전부증명서 확인
- 정관 최신본 확보
- 진행하려는 변경이 `1건인지`, `복합 사건인지` 구분
- 회사 형태, 이사회 설치 여부, 대표권 방식 확인
- 법정 신청기한 존재 여부 확인
- 관할 등기소 추정
- 전자신청 가능성 검토

### Step 2. 필수 정보 수집 및 원인행위 구조 확정
- 공통 필수 정보 수집
- 케이스 특화 변수 수집
- 결의기관 확정
  - 이사회
  - 주주총회
  - 사원총회
  - 대표이사 결정
  - 발기인 결정
- 필요 시 계약/동의/승낙/사임/인수 구조 설계

### Step 3. 증빙 및 첨부서류 확보
- 의사록, 정관, 주주명부, 취임승낙서, 사임서, 납입증명서, 위임장 등 확보
- 외국어 서류는 번역문 및 인증 필요성 검토
- 인감/전자증명서 상태 확인
- 세금 및 수수료 납부 전 준비 완료

### Step 4. 신청서 및 부속서류 작성
- 변경등기 신청서 작성
- 각 문서 간 필수 정보 일치 여부 점검
  - 성명
  - 주소
  - 날짜
  - 금액
  - 주식 수
  - 자본금
- 관할, 등록면허세, 첨부문서 리스트 확정

### Step 5. 등기 신청
- 전자신청 또는 서면신청
- 등록면허세 및 교육세 납부
- 등기신청수수료 납부
- 접수번호 확인
- 보정 가능성 높은 사건은 접수 직후 모니터링

### Step 6. 보정 대응 및 완료 후 후속조치
- 보정 통지 수신 여부 확인
- 보정 사유 분류
  - 결의기관 오류
  - 첨부서면 누락
  - 인적사항 불일치
  - 정관 특약 미반영
  - 인증/번역 문제
  - 세금/수수료 문제
- 보정 완료 후 등기 완료 여부 확인
- 후속조치 체크
  - 사업자등록 정정
  - 은행 정보 변경
  - 계약서/인허가 정보 변경
  - 내부 문서 및 날인체계 업데이트

## 4. 가격/ETA에 영향을 주는 변수

### 가격에 영향을 주는 변수
- 단일 사건인지 복합 사건인지
- 회사 형태가 주식회사인지 기타 회사인지
- 정관 최신본 유무
- 등기사항전부증명서 최신본 유무
- 이사회 설치 여부 및 결의구조 난이도
- 임원/주주 수
- 공동대표 여부
- 외국인/외국법인 관여 여부
- 외국어 문서 및 인증 필요 여부
- 자본금 증감 또는 주식 관련 계산 복잡도
- 채권자보호절차 필요 여부
- 합병/분할 등 조직행위 여부
- 과거 보정/반려 이력
- 전자신청 가능 여부
- 긴급도
- 서류 준비 상태

### ETA에 영향을 주는 변수
- 의사결정자 일정
- 결의기관 소집 기간
- 정관 확보 여부
- 인감/전자증명서 준비 상태
- 납입 또는 금융증빙 확보 속도
- 외국 서류 발급 및 인증 소요
- 채권자보호절차 기간
- 관외 이전 여부
- 복합 사건 여부
- 보정 발생 가능성

## 5. 퍼널 질문 설계안
전체 케이스를 대상으로 하는 `마스터 퍼널` 기준이다. 실제 운영에서는 응답에 따라 하위 시나리오로 분기해야 한다.

### Depth 1. 사건 분류용 질문
- Q1. 이번에 진행하려는 법인 등기 유형은 무엇인가요?
  - type: `multi_choice`
  - 예시 옵션: `설립`, `상호 변경`, `목적 변경`, `본점 이전`, `임원 변경`, `자본금 증자`, `자본금 감소`, `합병/분할`, `해산/청산`, `기타`
  - 목적: 메인 시나리오 분기
- Q2. 회사 형태는 무엇인가요?
  - type: `single_choice`
  - 예시 옵션: `주식회사`, `유한회사`, `합명회사`, `합자회사`, `외국회사`, `기타`
  - 목적: 법적 구조 분기
- Q3. 이번 건은 한 가지 변경만 있나요, 여러 변경을 한 번에 진행하나요?
  - type: `single_choice`
  - 예시 옵션: `단일 사건`, `복합 사건`
  - 목적: 가격/ETA 가산
- Q4. 희망 완료일은 언제인가요?
  - type: `date`
  - 예시: `2026-05-20`
  - 목적: 긴급도 산정
- Q5. 최신 정관을 보유하고 있나요?
  - type: `single_choice`
  - 예시 옵션: `있음`, `없음`, `불확실`
  - 목적: 문서 준비도 확인

### Depth 2. 절차/난이도 분기 질문
- Q6. 이사회 설치회사인가요?
  - type: `single_choice`
  - 목적: 결의기관 분기
- Q7. 외국인 또는 외국법인이 관련되어 있나요?
  - type: `single_choice`
  - 목적: 번역/인증 분기
- Q8. 법인인감 및 전자신청 수단을 보유하고 있나요?
  - type: `single_choice`
  - 목적: 신청 방식 분기
- Q9. 과거 동일 건으로 보정 또는 반려된 적이 있나요?
  - type: `single_choice`
  - 목적: 리스크 가산
- Q10. 이번 건에 임원 변경이 포함되나요?
  - type: `single_choice`
  - 목적: 임원 하위 퍼널 진입
- Q11. 이번 건에 자본금 또는 주식 수 변경이 포함되나요?
  - type: `single_choice`
  - 목적: 자본 하위 퍼널 진입
- Q12. 본점 주소가 관할 등기소를 넘어 이동하나요?
  - type: `single_choice`
  - 목적: 관내/관외 분기
- Q13. 채권자보호절차가 필요한 구조인가요?
  - type: `single_choice`
  - 목적: 조직행위/감자/해산 하위 분기

### Depth 3. 상세값 수집 질문
- Q14. 법인등기부상 정확한 법인명은 무엇인가요?
  - type: `text`
- Q15. 법인등록번호는 무엇인가요?
  - type: `text`
- Q16. 현재 본점 주소는 무엇인가요?
  - type: `text`
- Q17. 대표자 성명, 생년월일, 주소는 무엇인가요?
  - type: `text`
- Q18. 실무 연락 담당자의 이름, 휴대전화, 이메일은 무엇인가요?
  - type: `text`
- Q19. 임원 변경이 있다면 대상 직위와 변경 유형은 무엇인가요?
  - type: `multi_choice`
- Q20. 임원 변경 효력일은 언제인가요?
  - type: `date`
- Q21. 자본금 증감이 있다면 변경 전/후 자본금은 얼마인가요?
  - type: `number`
  - 단위: `원`
- Q22. 신규 발행 또는 감소되는 주식 수는 몇 주인가요?
  - type: `number`
  - 단위: `주`
- Q23. 주금 납입일 또는 기준일은 언제인가요?
  - type: `date`
- Q24. 관련 의사록, 정관, 주주명부, 납입증명서 중 현재 보유한 서류는 무엇인가요?
  - type: `multi_choice`
- Q25. 해외 체류자, 외국어 서류, 공동대표, 현물출자 등 특이사항이 있나요?
  - type: `multi_choice`

## 6. 추가로 확인해야 할 불확실한 포인트
- `모든 케이스`의 정확한 운영 범위를 어디까지 볼 것인지
  - 추정/불확실: 주식회사만 우선할지, 유한회사·외국회사·비영리법인까지 포함할지 확정 필요
  - 확인 질문: `우선 운영 범위를 주식회사로 한정할까요, 아니면 다른 회사형태까지 동시에 포함할까요?`
- 등기 신청 주체를 어디까지 지원할 것인지
  - 추정/불확실: 고객 셀프 작성 보조인지, 파트너 대리신청 전제인지에 따라 질문과 산출물이 달라짐
  - 확인 질문: `이 문서는 셀프 진행용인가요, 파트너/법무사 대리 진행용인가요, 둘 다인가요?`
- 가격 산정의 실제 기준
  - 추정/불확실: 내부 기본가와 가산 규칙 수치가 없으므로 현재는 변수만 정리
  - 확인 질문: `내부 기본가와 가산 단가 표가 이미 있나요?`
- ETA 기준점
  - 추정/불확실: `문의일 기준`인지 `필수 서류 수집 완료일 기준`인지에 따라 안내가 달라짐
  - 확인 질문: `ETA는 문의 접수 시점부터 계산하나요, 서류 완비 시점부터 계산하나요?`
- 전자신청 범위
  - 추정/불확실: 회사별 전자증명서, 인감카드, 서명환경에 따라 전자신청 가능 여부가 달라짐
  - 확인 질문: `전자신청을 기본값으로 둘까요, 불가 시 서면으로 전환하는 구조로 설계할까요?`
- 관할별 차이 반영 수준
  - 추정/불확실: 일반 기준은 통합 가능하지만, 일부 관할 보정 실무는 내부 사례 데이터가 필요
  - 확인 질문: `관할별 보정사례 DB나 내부 노트가 이미 있나요?`
- 자본·조직행위 케이스의 깊이
  - 추정/불확실: 합병, 분할, 현물출자, 전환사채 등은 별도 전문 시나리오로 분리하는 것이 일반적
  - 확인 질문: `합병/분할/현물출자 같은 고난도 케이스도 이번 마스터 퍼널 안에 포함할까요, 별도 전문 시나리오로 분리할까요?`
