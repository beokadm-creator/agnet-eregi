# 개발 구현 상황 정리

작성일: 2026-05-02

## 1. 전체 판단

현재 저장소는 초기 골격 수준이 아니라, 실제 서비스 운영을 전제로 한 멀티앱/백엔드 모노레포 구조까지 구현된 상태입니다.

- 상태 요약: 주요 기능 대부분 구현됨
- 현재 단계: 기능 확장보다 안정화/고도화/정리 단계에 가까움
- 최근 작업 방향: AI 기능 강화, 프롬프트 보안 보강, 콘솔/웹 UI 레이아웃 정리

## 2. 저장소 구조 확인 결과

핵심 구성은 다음과 같습니다.

- `firebase-react/apps/user-web`
  - 사용자 웹 앱
- `firebase-react/apps/partner-console`
  - 파트너 콘솔
- `firebase-react/apps/ops-console`
  - 운영 콘솔
- `firebase-react/apps/user-mobile`
  - 모바일 앱
- `firebase-react/functions`
  - Firebase Functions 기반 API/워커/배치

루트 `README.md` 기준으로는 프로젝트가 `Phase 11 최종 완료` 상태로 문서화되어 있습니다.

## 3. 구현 완료로 판단되는 영역

### 사용자 웹

다음 페이지/구성 요소가 확인되었습니다.

- 퍼널: `Funnel.tsx`
- 퍼널 결과: `FunnelResults.tsx`
- 대시보드: `Dashboard.tsx`
- 제출 상세: `SubmissionDetail.tsx`
- 설정: `Settings.tsx`
- 파트너 신청: `PartnerApply.tsx`
- 챗 위젯: `FloatingChatWidget.tsx`
- 에러 처리/404: `ErrorBoundary.tsx`, `NotFound.tsx`
- 다국어 리소스: `public/locales/en`, `public/locales/ko`

### 파트너 콘솔

아래 기능군이 실제 화면/컴포넌트 레벨로 존재합니다.

- 케이스 목록/상세
- 견적 관리
- 증빙 검토
- 환불 처리
- 패키지 관리
- 워크플로 전이
- 템플릿 관리
- 팀 멤버/조직/보안/웹훅/알림 설정
- 구독/정산/엔터프라이즈 분석

### 운영 콘솔

운영 기능도 별도 콘솔로 폭넓게 구현되어 있습니다.

- 운영 대시보드
- 인시던트 관리
- SLA 대시보드
- 관측성
- 감사 로그
- 리포트
- 릴리즈
- 접근 제어
- 시스템/설정
- 파트너 신청 검토
- 케이스 팩 관리

### 모바일 앱

Expo 기반 모바일 구조가 존재하며 아래 흐름이 구현되어 있습니다.

- 홈/프로필/설정
- 케이스 목록 및 상세
- 요청 상세
- 문서/이미지 업로드
- 알림/푸시 등록 관련 구성

### 백엔드

Functions 쪽은 단순 API 몇 개 수준이 아니라, 매우 넓은 범위가 구현되어 있습니다.

- `v1` 라우트 다수 존재
- 사용자/파트너/운영/결제/정산/문서/승인/리포트/B2B/B2G 관련 API 포함
- 알림/정산/구독/리텐션/SLO/백업/예측/문서AI 등 배치 워커 존재

라우트 수 기준으로도 상당한 폭의 API가 확인되었습니다.

## 4. 최근 커밋 흐름

최근 커밋 메시지 기준 주요 흐름은 다음과 같습니다.

## 6. 검증 결과

### 빌드

`firebase-react`에서 `npm run build` 실행 결과:

- `functions` 빌드 진행됨
- 공통 패키지 빌드 진행됨
- `user-mobile` 타입 기반 빌드 진행됨
- `user-web`, `partner-console`, `ops-console`는 빌드 실패

실패 원인은 코드 자체보다 환경 의존성 누락으로 보입니다.

- 누락 모듈: `@rollup/rollup-linux-arm64-gnu`

즉 현재 프론트 빌드 실패는 구현 불가라기보다, 로컬 설치 상태 문제일 가능성이 큽니다.

### 린트

`npm run lint` 결과:

- 프론트/모바일/공통 패키지 TypeScript 체크는 통과
- `functions` 린트는 실패

실패 원인:

- `eslint: not found`

이 역시 코드 품질 문제보다 개발 의존성 설치 문제로 해석됩니다.

### 테스트

`functions` 테스트 결과:

- 테스트 스위트: 19개 중 18개 통과
- 테스트 케이스: 81개 중 80개 통과
- 실패 1건:
  - `src/__tests__/routes/quotes.test.ts`
  - `POST /v1/partner/cases/:caseId/quotes/draft`

## 7. 현재 드러난 주요 이슈

### 이슈 1. 프론트 빌드 환경 불안정

- Rollup optional dependency 누락으로 프론트 앱 빌드 불가
- 코드 이상보다 설치 상태 문제로 보임

### 이슈 2. Functions lint 환경 미정리

- `eslint` 실행 파일 부재
- `functions/package.json`에 lint 스크립트는 있으나 실제 dev dependency 정합성 점검 필요

### 이슈 3. Quotes 테스트 1건 실패

해당 테스트는 타임아웃으로 실패했습니다.

가능성 높은 원인:

- `quotes` draft 생성 라우트에 환율 조회/변환 로직이 추가됨
- 테스트에서 이 외부/비동기 의존이 충분히 목킹되지 않음
- 구현 부재보다는 테스트와 실제 로직 간 불일치 가능성이 높음

## 8. AI 관련 상태

별도 문서와 변경 파일 기준으로 AI 기능은 실제 여러 지점에 연결되어 있습니다.

- 유저 퍼널 추천 생성
- 제출 상세 AI 보조
- 유저 챗봇
- 파트너 증빙 검토/초안 생성
- 운영 인시던트 triage
- 번역/프롬프트 안전화

`AI_SCENARIOS.md`와 메모 파일을 보면 현재는 단순 호출 연결을 넘어 아래 항목까지 다루고 있습니다.

- 사용 지점 정리
- 일일 쿼터 제한
- 프롬프트 하드닝
- 입력 sanitize

## 9. 종합 판단

현재 개발 상태는 다음처럼 보는 것이 가장 정확합니다.

- 기능 구현 수준: 높음
- 제품 완성도: 프로덕션 직전 또는 내부 운영 가능 수준
- 현재 남은 일: 신규 대형 기능보다 안정화/검증/환경 복구/잔여 수정

즉,

**“대부분 구현 완료 + 현재는 AI 고도화와 화면 정리, 테스트/빌드 안정화 작업 진행 중”**

이라고 정리할 수 있습니다.

## 10. 우선순위 제안

다음 순서로 정리하는 것이 효율적입니다.

1. 프론트 빌드 환경 복구
   - Rollup optional dependency 재설치
2. `functions` lint 환경 복구
   - eslint 의존성 정리
3. `quotes` 테스트 실패 원인 보정
   - 환율 관련 mock 또는 timeout 원인 수정
4. 현재 미커밋 변경 검토 후 커밋 단위 정리
   - AI 보강
   - UI 레이아웃 조정
   - 보안/프롬프트 sanitize

## 11. 실무용 한 줄 결론

현재 프로젝트는 “개발 중인 데모” 수준이 아니라, **주요 기능이 넓게 구현된 운영형 플랫폼**이며, 지금은 **마무리 안정화와 고도화 단계**로 보는 것이 맞습니다.
