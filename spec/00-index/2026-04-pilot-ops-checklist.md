# 파일럿 운영 체크리스트 (corp_officer_change_v1)

작성일: 2026-04-17  
대상: 파일럿 기간 중 발생하는 `corp_officer_change_v1` 케이스 운영 대응 가이드

---

## 1) 모니터링 필수 이벤트 (타임라인 핵심 로그)
파일럿 기간 동안 Ops 담당자는 케이스 타임라인에 다음 이벤트들이 정상적으로 찍히는지 확인합니다.
- `WORKFLOW_STAGE_CHANGED`: 각 단계 전진 여부 파악 (지연 시 이유 확인)
- `DOCUMENT_REVIEWED_NEEDS_FIX`: 사용자의 서류 보완이 필요한 경우 (User Web 알림 확인)
- `SIGNATURE_TASK_COMPLETED`: 서명본 제출 및 승인 완료 확인
- `PACKAGE_READY`: 제출 패키지 생성 완료 여부 (다운로드 가능 신호)
- `CASE_STATUS_CHANGED`: 전체 상태가 `completed`로 정상 종료되었는지 확인

*참고: 타임라인에서 시간이 비정상(`Invalid Date` 혹은 빈칸)으로 보인다면 이벤트 객체 내에 `occurredAt`이 정상적으로 Timestamp나 ISO string 형태로 기록되어 있는지 Firestore 문서를 직접 확인하세요.*

## 2) 리스크 및 수동 복구 포인트

### A. 서류 누락으로 인한 자동 전진 불가
- **증상**: 파트너가 서명본을 다 올린 것 같은데 `filing_submitted`로 전진이 안 됨.
- **원인**: 케이스 조건(사임 임원 여부 등)에 따라 요구되는 `requiredSlots` 중 미승인(needs_fix/missing) 상태의 문서가 남아있음.
- **수동 복구**: 
  1. Ops 콘솔(또는 Firestore 직접 확인)에서 해당 케이스의 `documents` 컬렉션 중 `status !== 'ok'`인 필수 슬롯을 찾음.
  2. 수동으로 해당 문서를 `review ok` 처리.

### B. 외부 연동 및 네트워크 오류 발생
- **증상**: 폼 저장, 위젯 업로드 중 네트워크 에러 혹은 Functions 에러(500) 발생 시, 상단에 에러 배너나 패널 내에 인라인 에러 박스가 나타남.
- **원인**: Storage 토큰 만료, Functions Cold Start 타임아웃, 오타/데이터 형식 오류 등.
- **수동 복구**:
  1. UI에 표시된 붉은색 에러 메시지를 확인.
  2. "다시 시도" 버튼을 클릭해 재요청. 계속 실패할 경우 타임라인 로그의 직전 성공 이벤트를 확인하여 복구 시점 특정.

### C. 제출 패키지(PACKAGE_READY) 미생성 혹은 다운로드 오류
- **증상**: 케이스가 `completed`가 되었는데 타임라인에 `PACKAGE_READY`가 없고 다운로드 버튼이 보이지 않거나, 다운로드 버튼 클릭 시 에러가 발생함.
- **원인**: 백엔드의 문서 병합(ZIP/DOCX) 워크플로우 큐가 지연되거나 실패함. 혹은 이벤트 `meta`에 `submissionPackagePath` 및 `closingReportPath` 경로가 누락/잘못 기재됨.
- **수동 복구**:
  1. 타임라인 이벤트의 `meta` 데이터를 검사해 경로가 올바른지 확인 (`v1/cases/...` 형태).
  2. 패키지가 없다면 Ops 환경에서 제공되는 패키지 수동 생성 스크립트 실행.

## 3) 파트너 문의 응대 가이드
- **"다음 단계로 넘어가는 버튼이 비활성화되어 있어요"**
  - **대응**: "버튼 위에 마우스를 올리시거나 화면 상단 배너를 보시면 '사유(reasonKo)'가 나와 있습니다. 누락된 서명본이나 필수 서류를 확인해주세요."
- **"업로드 중에 실패했다고 나옵니다"**
  - **대응**: "일시적인 네트워크 오류일 수 있습니다. 서명본/접수증 패널에서 '다시 시도'를 클릭해주세요."

## 4) 일일/주간 운영 루프 (Gate 검증 및 백로그 생성)
파일럿 기간 동안 매일/매주 아래 절차에 따라 Ops Console을 활용하여 검증 결과를 기록하고 스프린트 백로그를 산출합니다. (로컬 스크립트 실행 불필요)

### [매일 1회] 오늘 운영 요약 확인 및 기록
1. Ops Console에 접속하여 **"오늘 운영 요약"** 패널을 확인합니다.
2. 날짜를 선택(기본 오늘)하면 일일 Gate 집계와 백로그 후보가 한 화면에 표시됩니다.
3. [일일 로그용 복사] 버튼을 눌러 일일 Gate 집계 결과를 **일일 운영 로그**에 붙여넣습니다.
4. 실패 케이스가 있다면 "케이스 1페이지" 섹션에 `caseId`를 입력하고 상세 내역(Case/Docs/Timeline/Gate Evidence)을 한 화면에서 확인하며 트러블슈팅합니다.

### [주 1회] 스프린트 백로그 이관
1. 주 1회 스프린트 플래닝 시 Ops Console의 "오늘 운영 요약" 패널에서 기간을 확인합니다.
2. [스프린트 백로그용 복사] 버튼을 눌러 생성된 마크다운을 **이슈 트래커(스프린트 백로그)**에 그대로 등록합니다.
