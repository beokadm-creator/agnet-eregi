# UX 스펙(개발용) — 모바일 사용자 퍼널 + PC 업무 콘솔

## 1) 핵심 목표(KPI로 정의)

사용자가 “업무명을 몰라도” 끝까지 가게 만드는 UX의 목적은 아래 3개 KPI로 요약됩니다.

- **입구 마찰 최소화**: 홈 → 진단 시작률(5초 이탈률 감소)
- **진단 마찰 최소화**: 진단 완료율(질문 수 3~7개), 문항당 이탈률 감소
- **선택 피로 최소화**: 파트너 선택률(추천 1안 수락률), 비교 클릭률/스크롤 깊이 최적화

추가 운영 KPI:
- **TTFU(Time to First Upload)**: 파트너 선택 → 첫 서류 업로드까지 시간  
- **보완요청률**: 문서 품질/체크리스트 품질을 역으로 측정  
- **SLA 준수율**: 파트너/플랫폼 운영 품질 지표

---

## 2) 듀얼 UX 전략(정답: 사용자=모바일, 일=PC)

등기 업무는 “구매/선택”은 모바일이 쉽고, “서류 검토/제출”은 PC가 유리합니다. 따라서 제품을 3개 면으로 분리합니다.

1. **모바일 사용자 앱/PWA**: 검색(자연어) → 미니 진단 → 추천/비교 → 사건 진행(알림 중심)
2. **PC 파트너 콘솔**: 케이스 큐 → 문서 슬롯 검토 → 견적 확정(승인 로그) → 제출/완료
3. **PC 운영 콘솔(Ops)**: 수동검토 큐, SLA 브리치, 분쟁/환불/정산, 광고/노출 정책 관리

---

## 3) 모바일 사용자 퍼널(화면 단위)

### 3.1 Home(구글형 입구)

**원칙**: “자연어 1줄 + 카테고리 칩” 외에는 아무것도 강제하지 않는다.

- 입력창(placeholder): “무엇을 하시려나요? 편하게 적어보세요”
- 예시(클릭하면 바로 입력): 대표 변경 / 본점 이전 / 담보 설정
- 카테고리 칩: 법인등기 / 기타등기(동산·채권담보)
- 금지: 로그인 강제, 업로드 강제, 상세 폼 강제

**엔진 연동(개요)**: `intent_router`가 사건팩 후보(1~3개) + 첫 질문을 만든다.

### 3.2 Mini Diagnosis(3~7 문항)

**원칙**: 질문은 분기 트리 필수만. 대신 즉시 “가치”를 제공한다.

- 질문 카드: 1화면 1문항(최대 7개)
- 동시에 보이는 가치 프리뷰 카드:
  - 예상 비용(범위) / 예상 소요시간(범위) / 준비물 Top3(임시)
  - “확정 아님” 라벨 + 전제조건(정보 미확정으로 범위 넓음) 표시

**이탈 방지**:
- “나중에 이어하기”를 기본(자동 저장 + 딥링크 재진입)
- 질문 도중에도 결과 미리보기(“현재 기준으로는 임원변경 가능성이 높아요”)

### 3.3 Results(파트너 결과)

**원칙**: 리스트를 길게 보여주면 선택 피로로 이탈한다. 기본은 **추천 1안 + 비교 Top3**.

- 상단: 추천 파트너 1안(근거 1~3줄: 적합도/SLA/품질/가용성)
- 하단: 비교 리스트 Top3 + “더 보기”
- 스폰서(광고): 별도 영역(캐러셀) + “광고” 라벨 고정

**카드 필수 정보(비교 가능성)**:
- 예상 총액(범위) + 구성(실비/보수/플랫폼 이용료)
- ETA 범위
- 방문 가능 배지(지도)
- 응답속도/SLA 요약, 리뷰/완료율 요약

### 3.4 Partner Detail(직방/다방 감성)

**원칙**: “방문 가능”은 옵션이지만, 카드/지도 정보는 표준화한다.

- 지도/주소/영업시간/주차/예약 버튼
- 온라인 진행 CTA가 primary, 방문 예약은 secondary
- 책임 분리: “플랫폼이 제공”처럼 보이지 않게 “파트너 방문”임을 명시

### 3.5 Case Workspace(사건 진행 OS)

탭: 요약 / 준비물 / 업로드 / 진행 / 메시지 / 결제

- 서류는 **슬롯 단위**로 상태 표시(미업로드/검토중/OK/보완필요)
- AI 검토는 “초안/추천”만 노출하고, 보완 요청 메시지는 **파트너/운영 승인 후** 발송
- 재진입 알림: 다음 할 일, 마감, SLA 브리치, 보완 요청 등을 푸시/카톡으로 안내

## 4) PC 파트너 콘솔(업무 처리 UX)

### 4.1 케이스 큐(목록)

- 필터: 사건팩/긴급/마감/SLA/상태
- 빠른 액션: 수락/거절/보완요청/견적확정
- SLA 타이머: 남은 시간 + 브리치 경고(색상)

### 4.2 케이스 상세

- 좌측: 문서 슬롯(드래그&드롭 업로드 + 버전 관리 + 슬롯 추천)
- 우측: 체크리스트 + 메시지 + AI 초안(승인 필요)

### 4.3 견적 확정(승인 로그)

- 견적은 “범위 제안(자동)” → “확정(사람)” 2단계
- 확정 후: 사용자 동의(추가비용/추가서류/마감) 카드 발송
