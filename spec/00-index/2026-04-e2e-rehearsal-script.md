# 파일럿 운영 리허설 스크립트 (corp_officer_change_v1 E2E)

작성일: 2026-04-17  
대상: 임원변경 등기(corp_officer_change_v1) 케이스를 파트너 콘솔에서 처음부터 끝까지 처리하는 1건의 End-to-End 데모 시나리오

---

## 1) 준비 (Intake ~ Docs Collect)
1. **파트너 로그인**: 콘솔에 접속하여 "익명 로그인" 후 "dev: partner 전환(p_demo_01)" 클릭
2. **케이스 큐 확인**: 좌측 패널의 `케이스 큐`에서 상태가 `new`인 케이스를 찾습니다.
3. **수락 및 작업 시작**: `수락` 버튼 클릭 -> `in_progress` 상태로 변경 후 `작업 열기` 클릭

---

## 2) 문서 검토 (docs_review)
- **목표**: 사용자가 제출한 기초 필수 서류들을 승인합니다.
- **성공 신호**:
  - `문서 검토` 패널에서 문서 선택 후 **판정: ok** 저장
  - 타임라인에 `DOCUMENT_REVIEWED_OK` 이벤트 발생
  - 모든 필수 문서가 `ok`가 되면, 상단 상태바의 "다음 단계 진행 가능" 문구가 초록색으로 활성화
- **액션**: `다음 단계로` 버튼 클릭 (Stage가 `draft_filing`으로 변경됨)
- **타임라인 신호**: `WORKFLOW_STAGE_CHANGED` (docs_review → draft_filing) 배지 발생
*(참고: 필수 문서들이 모두 자동 통과 조건을 만족하는 경우 이 과정 없이 바로 `draft_filing`으로 넘어올 수 있습니다.)*

---

## 3) 초안 작성 및 서명 확보 (draft_filing)
- **목표**: 임원변경 정보를 입력해 서류를 자동 생성하고 서명본을 업로드합니다.
- **액션**:
  1. `회사명`, `결의/안건`, `임원 추가` 등의 정보를 입력하고 **정보 저장**
  2. **의사록/위임장/신청서** 등 자동 생성 버튼 클릭 (다운로드 가능)
  3. **서명본 업로드**: 하단의 위젯에서 누락된 서명본(예: 의사록, 위임장) 파일을 선택해 **업로드 및 확정** 클릭
- **성공 신호**:
  - 서명본 1개 업로드 시마다 타임라인에 `SIGNATURE_TASK_COMPLETED` 배지 발생
  - 모든 필수 서명본이 업로드 완료되면, 누락 알림이 초록색 `✅ 확보 완료`로 바뀜
- **다음 단계 진행**:
  - 조건이 모두 충족되면 서버에서 **자동으로 `filing_submitted`로 전진**시킵니다. (이 경우 수동 버튼 클릭 생략)
  - 자동 전진되지 않는 경우, 상단 상태바가 활성화되었을 때 `다음 단계로` 버튼 클릭
- **타임라인 신호**: `WORKFLOW_STAGE_CHANGED` (draft_filing → filing_submitted) 배지 발생

---

## 4) 등기 제출 및 접수증 (filing_submitted)
- **목표**: 관할 등기소에 제출한 정보를 입력하고 접수증을 올려 케이스를 완료합니다.
- **액션**:
  1. 접수번호, 관할(예: 서울중앙지방법원), 접수일, 메모 입력 후 **저장**
  2. 접수증 위젯에서 파일을 선택해 **업로드 및 확정** 클릭
- **성공 신호**:
  - 접수증 업로드 성공 및 자동 Review OK
  - 타임라인에 `DOCUMENT_REVIEWED_OK` (접수증)
- **다음 단계 진행**:
  - 접수 정보와 접수증이 모두 확인되면 서버에서 **자동으로 `completed`로 완료** 처리합니다. (수동 버튼 클릭 생략)
  - 자동 전진되지 않는 경우 활성화된 버튼 클릭
- **타임라인 신호**: 
  - `WORKFLOW_STAGE_CHANGED` (filing_submitted → completed)
  - `CASE_STATUS_CHANGED` (completed)

---

## 5) 케이스 완료 및 다운로드 (completed)
- **목표**: 케이스 최종 종료 및 제출 패키지 획득
- **성공 신호**:
  - 상단 패널에 초록색 `✅ 케이스 완료 (completed)` 배너 노출
  - 타임라인에 `PACKAGE_READY` 배지 발생
  - **다운로드 2개 버튼** (제출 패키지 ZIP, 종료 리포트 DOCX) 활성화
- **액션**: 버튼을 클릭하여 정상적으로 파일 다운로드가 이루어지는지 최종 확인
