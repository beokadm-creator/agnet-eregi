# 2주 스프린트 PRD (뼈대 먼저 + 모듈화)

작성일: 2026-04-17  
대상: 임원변경 등기(corp_officer_change_v1) “파일럿 운영 가능한 MVP”를 **2주 내** 제품 형태로 정리

---

## 1) 목적(Why)
현재 시스템은 핵심 엔진(케이스/워크플로우/문서/서명본/자동전진/제출패키지)이 연결되어 있으나, 화면/운영 플로우가 “데모형 조립”에 가까워 **실사용자가 매일 쓰는 프로덕트 뼈대**가 부족함.  
이번 스프린트의 목적은:

1. “일 흐름” 관점에서 **파트너가 케이스를 끝까지 처리하는 작업보드**를 완성한다.  
2. 기능을 모듈 단위로 정리하여(모듈화) 다음 케이스팩/채널 확장 시 재사용성을 확보한다.  
3. 파일럿 운영에 필요한 최소 운영 기능(가시성/감사/오류 대응)을 갖춘다.

---

## 2) 범위(Scope)

### In Scope (이번 2주에 반드시)
**A. 파트너 콘솔 “작업보드” 뼈대**
- 케이스 리스트(검색/필터/상태별 큐) + 케이스 상세 “작업 흐름” 통합 화면
- 작업 흐름을 단계별 카드로 고정:
  - docs_review → draft_filing → filing_submitted → completed
- 각 단계에서 “해야 할 일”이 자동으로 드러나게:
  - 필수서류/서명본 누락표(체크리스트) + 업로드/검토 액션
  - 자동 전진/완료 상태 표시 + 완료 시 다운로드 CTA

**B. 모듈화(구조 정리)**
- UI: “케이스 작업보드 모듈”, “문서/서명본 모듈”, “제출/완료 모듈”로 컴포넌트 분리
- API: 이미 있는 엔드포인트는 유지하되, 화면이 의존하는 데이터 흐름을 일관된 형태로 래핑(프론트 유틸/훅)

**C. 파일럿 운영 최소 요건**
- 타임라인 기반의 “왜 지금 이 상태인지” 가시화(이미 이벤트 있음 → UI에서 더 활용)
- 다운로드: 제출패키지/종료리포트/서류 DOCX/서명본 포함 ZIP
- 기본 오류 대응 UX(실패 메시지, 재시도, 누락 사유 reasonKo 노출)

### Out of Scope (이번 스프린트에서 보류)
- PG/정산 실연동(구조는 유지, 운영은 다음 단계)
- 멀티 케이스팩(이번은 corp_officer_change_v1 중심)
- 외부 알림(이메일/SMS/카톡) “실발송” (단, 이벤트 훅/포인트 설계는 남김)

---

## 3) 사용자/역할
- **Partner(등록 법무사)**: 케이스를 처리하는 주 사용자
- **Ops(운영자)**: 예외/분쟁/재처리 담당
- **User(의뢰인)**: 서류 제출/보완, 진행 확인, 완료 후 파일 다운로드

---

## 4) 사용자 플로우(Partner 기준)
1. 케이스 큐에서 새 케이스 선택
2. docs_review: 필수서류 상태 확인 → needs_fix면 보완 사유 처리
3. draft_filing:
   - 입력 폼 확인/수정
   - 템플릿 생성/다운로드
   - 서명본 업로드/검토 OK → 자동으로 filing_submitted로 전진(이미 구현됨)
4. filing_submitted:
   - 접수정보 입력 + 접수증 업로드/검토 OK → 자동 completed(이미 구현됨)
5. completed:
   - 제출 패키지(ZIP)/종료리포트(DOCX) 다운로드

---

## 5) 성공 지표(Success Metrics)
- Partner가 1건 케이스를 **10분 내**(데이터 준비 기준) completed까지 처리 가능
- “다음 단계 진행 불가 사유(reasonKo)”가 UI에서 항상 확인 가능(추측 최소화)
- completed 이후 제출패키지/리포트 다운로드까지 1클릭

---

## 6) 기능 요구사항(요약)
### 6.1 Partner Console 작업보드
- 상단: stage, canAdvance, reasonKo, 핵심 CTA(다음 단계 / 다운로드)
- 중단: 단계별 섹션(문서/서명본/접수)
- 하단: 타임라인(특히 PACKAGE_READY 등 핵심 이벤트는 액션 버튼 포함)

### 6.2 User Web(최소)
- 케이스 상태/타임라인 로드
- completed/PACKAGE_READY 감지 시 다운로드 CTA

---

## 7) 수용 기준(Acceptance Criteria)
- corp_officer_change_v1 1건을 로컬/에뮬레이터 환경에서 end-to-end로:
  - docs_review → draft_filing(서명본 ok) → filing_submitted(접수증 ok + 접수정보) → completed
  - completed에서 제출패키지 ZIP 다운로드 가능
- partner-console에서 “해야 할 다음 액션”이 항상 1개 이상 표시(없으면 완료 상태)
